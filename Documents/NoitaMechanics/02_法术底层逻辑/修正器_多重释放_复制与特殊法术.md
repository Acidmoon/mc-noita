# 修正器、多重释放、复制与特殊法术

## 修正器的基本模式

多数 `ACTION_TYPE_MODIFIER` 逻辑是：

1. 改写 `c` 的一个或多个字段。
2. 调用 `draw_actions(1, true)` 把下一张 action 纳入当前 shot。

例如伤害、速度、散射、暴击、穿透、爆炸、轨迹、状态附加等都属于“修改当前 shot 状态”。如果后面没有投射物，修正器通常不会单独产生可见输出。

## 多重释放的基本模式

`BURST_2`、`BURST_3`、`BURST_4`、`BURST_8` 等 `ACTION_TYPE_DRAW_MANY` 会调用 `draw_actions(n, true)`。它们不只是“同时发射 n 发”，而是把后续 n 张 action 都纳入同一 shot；其中可能又包含修正器、触发器或更多多重释放。

实现时要把多重释放当成抽牌控制流，而不是简单复制当前 projectile。

## 复制与希腊字母法术

`ACTION_TYPE_OTHER` 中有大量特殊法术，典型包括：

- 复制下一张、上一张、随机张。
- 复制使用次数有限的法术。
- 希腊字母类从 deck、discarded 或已施放记录中引用别的法术。
- 条件法术根据敌人、血量、金币、液体等环境决定继续抽哪段。
- 终结类法术会大范围加载实体或调用引擎事件。

这些法术的共同点是：它们操作的是“法术列表和施法上下文”，不是投射物实体本身。

## Wand Refresh

`RESET` 属于 `ACTION_TYPE_UTILITY`，但底层更像状态机操作。它把 `hand` 和 `deck` 清入 `discarded`，再在允许时把 `discarded` 重新排序回 deck。它还减少 `current_reload_time`。这使它可以改变同一轮施法里后续卡牌是否可再次被抽到。

## 实现优先级

MC-noita 初期可以分三层推进：

1. 先实现普通修正器：伤害、散射、速度、施放延迟、充能时间、额外实体。
2. 再实现多重释放和触发器，因为它们要求解释器支持嵌套 shot。
3. 最后实现复制、希腊字母、条件和 Wand Refresh；这些最容易破坏状态机，需要测试覆盖。
