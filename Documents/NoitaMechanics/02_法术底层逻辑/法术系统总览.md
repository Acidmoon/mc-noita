# 法术系统总览

> Noita 的“法术”在底层首先是 `gun_actions.lua` 中的一条 action 记录，再通过 XML 实体、Lua 脚本和引擎组件表现为实际效果。

## Action 的基本结构

每个 action 通常包含：

- `id`：稳定标识，如 `LIGHT_BULLET`。
- `name` / `description`：本地化 key，对应 `common.csv`。
- `type`：8 种 `ACTION_TYPE_*` 之一。
- `mana`：抽取并执行时的法力变化。
- `max_uses`：有限次数；缺省或 `-1` 通常表示无限。
- `spawn_level` / `spawn_probability`：生成池。
- `related_projectiles`：关联实体 XML。
- `action = function()`：真正的底层行为。

## 8 种 action 类型

| 类型 | 本轮解析数量 | 实现含义 |
|---|---:|---|
| `ACTION_TYPE_PROJECTILE` | 122 | 生成移动投射物，或生成带触发器的投射物。 |
| `ACTION_TYPE_STATIC_PROJECTILE` | 45 | 生成区域、云、光束、场等静态或半静态效果。 |
| `ACTION_TYPE_MODIFIER` | 143 | 修改当前 shot 状态 `c`，通常继续抽下一张。 |
| `ACTION_TYPE_DRAW_MANY` | 14 | 多重释放，额外抽取多张卡。 |
| `ACTION_TYPE_MATERIAL` | 26 | 生成或转换材料。 |
| `ACTION_TYPE_OTHER` | 42 | 随机、复制、希腊字母、条件、终结类等特殊逻辑。 |
| `ACTION_TYPE_UTILITY` | 25 | 传送、刷新、工具、召唤等功能性逻辑。 |
| `ACTION_TYPE_PASSIVE` | 5 | 被动或持有时效果。 |

## `action()` 决定卡牌真正行为

同样是“投射物法术”，底层也可能很不同：

- 普通投射物：`add_projectile("...xml")`。
- 触发：`add_projectile_trigger_hit_world("...xml", n)`。
- 定时：`add_projectile_trigger_timer("...xml", delay, n)`。
- 失效触发：`add_projectile_trigger_death("...xml", n)`。
- 特殊法术：可能直接 `EntityLoad()`、改写 `deck`、扫描后续卡牌、改变 `mana` 或调用引擎事件。

因此，分类文档里的 `type` 只能给第一层语义；真正实现时必须看 `action_summary` 或原始 `action()`。

## 自动档案如何使用

- 查完整列表：`03_法术分类档案/00_分类总览.md`。
- 查机器可读数据：`00_来源与方法/法术动作索引_422项.csv`。
- 查单类法术：`03_法术分类档案/ACTION_TYPE_PROJECTILE_投射物法术.md` 等。

这些表能回答“有哪些法术、ID 是什么、类型是什么、法力和次数是多少、action 大致做什么”。复杂法术仍需打开 `_source_cache/noitadata/data/scripts/gun/gun_actions.lua` 逐段核对。
