# Noita 法术与法杖底层逻辑归档

> 归档日期：2026-06-07。目标是为 MC-noita 的 Minecraft/Fabric 实现提供可追溯的 Noita 机制参考。

## 目录结构

- `00_来源与方法/`：来源等级、缓存文件、全量 action 索引 CSV。
- `01_法杖底层逻辑/`：法杖施法状态机、抽牌/弃牌/包裹、法杖数值结算。
- `02_法术底层逻辑/`：法术 action 类型、触发/定时/递归、修正器、多重释放、实体 XML。
- `03_法术分类档案/`：从 `gun_actions.lua` 自动解析出的 422 个 action，按 8 个 `ACTION_TYPE_*` 分类归档。
- `04_实现参考/`：当前 MC-noita Java 实现与 Noita 原机制的映射建议。
- `99_缺失与待核实/`：缺失翻译、无法由公开资料确认的底层内容。
- `_source_cache/`：本轮缓存的 noitadata Lua/XML/CSV 原始资料，便于离线复核。

## 最重要的结论

Noita 的法杖不是简单地“从左到右发射法术”。公开 datamine 显示，它有 `deck`、`hand`、`discarded` 三组卡牌状态，有一个会被修正器持续改写的当前施法状态 `c`，并通过 `draw_actions()`、触发器和特殊法术把后续卡牌嵌进当前 shot。

全量法术档案以 `gun_actions.lua` 为准：本轮解析到 422 个 action，其中投射修正法术最多，共 143 个；投射物法术 122 个；静态投射物 45 个；其余为多重释放、材料、实用、被动和特殊法术。

## 快速阅读顺序

1. 先看 `01_法杖底层逻辑/法杖施法状态机.md`。
2. 再看 `02_法术底层逻辑/法术系统总览.md`。
3. 需要具体法术时，查 `03_法术分类档案/` 或 `00_来源与方法/法术动作索引_422项.csv`。
4. 做 MC-noita 实现时，最后看 `04_实现参考/MC_Noita实现映射建议.md`。
