# 法杖施法状态机

> 核心来源：noitadata `data/scripts/gun/gun.lua`、`gun_enums.lua`，以及 Wiki 的 Wand Mechanics 系列页面。

## 全局状态

`gun.lua` 在一次施法过程中维护几类核心状态：

| 状态 | 作用 |
|---|---|
| `deck` | 当前还可抽取的法术卡牌。非乱序法杖按 `deck_index` 排序，乱序法杖在装填时洗牌。 |
| `hand` | 本次 shot 已经抽到并执行过的卡牌。结算后多数会进入弃牌堆。 |
| `discarded` | 已用过、暂时不能再抽的卡牌。法杖刷新或装填时会回到 `deck`。 |
| `c` | 当前 shot 的 `ConfigGunActionInfo`，修正器都在改写它。 |
| `current_reload_time` | 本轮结束后的充能时间，能被法术增减。 |
| `mana` | 当前施法上下文里的法力。负法力法术会增加它。 |
| `reloading` / `start_reload` | 表示本次抽牌是否已经触发装填。 |
| `recursion_limit` | Lua 中初始化为 2，用于限制部分递归调用。 |

## 一次施法的简化流程

1. 创建 shot，复制一份默认 `c` 状态。
2. 调用 `draw_actions(shot.num_of_cards_to_draw, instant_reload_if_empty)` 抽取法术。
3. 每抽到一个 action，先检查法力和使用次数，再执行 `action.action()`。
4. 投射物、静态投射物、材料法术会把 `got_projectiles` 标为真。
5. 如果 action 生成投射物，当前已激活的额外修正器会应用到投射物。
6. 抽牌结束后，`SetProjectileConfigs()` 把 `c` 和投射物配置传给引擎层。
7. 本次 `hand` 进入 `discarded`，有限次数法术在满足条件时消耗次数。

## 乱序与非乱序

`order_deck()` 是区分乱序与非乱序的关键：

- 乱序法杖：用当前帧作为随机种子洗牌；每轮装填后得到新顺序。
- 非乱序法杖：按卡牌原始 `deck_index` 排序，也就是玩家看到的槽位顺序。

这解释了为什么非乱序法杖可以做精确编排，而乱序法杖适合放同质投射物或不依赖顺序的组合。

## 法力与抽牌失败

`draw_action()` 在执行 action 前检查法力：

- action 未声明 `mana` 时，默认法力消耗为 `ACTION_MANA_DRAIN_DEFAULT=10`。
- 当前法力不足时，该 action 会进入 `discarded`，本次不会执行。
- `draw_actions()` 遇到抽牌失败后，会继续尝试从 `deck` 中抽后续可用 action。

这意味着“某个修正器没钱放不出来”不一定中断整个 shot；后续便宜法术仍可能被抽到并执行。

## 使用次数消耗

有限次数法术不是一抽到就必定扣次数。`move_hand_to_discarded()` 里只有在本次 shot 真的产生了投射物，或者 action 类型是 `OTHER` / `UTILITY` 等需要识别和记录的情况时，才会走使用次数减少逻辑。永久附着法术不会普通地进入弃牌堆。

## 与 MC-noita 的实现差异

当前 MC-noita 的 `NoitaWandCaster` 已有简化版状态：NBT 中保存 deck、draw index、施法延迟、充能、当前法力。它还没有完整模拟 Noita 的 `hand` / `discarded` / `c` 状态树，也没有 `draw_actions()` 递归抽牌模型。实现映射见 `04_实现参考/MC_Noita实现映射建议.md`。
