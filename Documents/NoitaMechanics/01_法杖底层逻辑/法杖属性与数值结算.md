# 法杖属性与数值结算

> 这里把玩家看到的法杖属性映射到底层结算位置。

## 核心属性

| 属性 | 底层影响 |
|---|---|
| 乱序 | 装填时是否洗牌 `deck`。 |
| 施放数 | 初始 `shot.num_of_cards_to_draw`，相当于每次 shot 预置多重释放。 |
| 施放延迟 | 本次 shot 后到下一次 shot 的等待时间，会再叠加法术的 `c.fire_rate_wait`。 |
| 充能时间 | `deck` 耗尽后进入装填的等待时间，会受 `current_reload_time` 影响。 |
| 最大法力 | 法杖可储存法力上限；预抽取 payload 时也会消耗。 |
| 法力充能速度 | 按时间恢复法力；法杖不在手上时仍可恢复。 |
| 容量 | 可放置法术的槽位数，即初始 deck 的大小上限。 |
| 散射 | 与法术、修正器的 `c.spread_degrees` 加总后影响发射偏角。 |
| 速度倍率 | 影响投射物初速，许多天然法杖为 1。 |
| 始终施放 | 作为永久附着 action 参与每次施法；通常不消耗普通法力和次数。 |

## `c` 状态的叠加方式

大多数法术不是直接修改法杖，而是修改当前 shot 的 `c`：

- `c.fire_rate_wait`：本次射击后施放延迟。
- `c.spread_degrees`：散射。
- `c.speed_multiplier`：投射物速度倍率。
- `c.damage_projectile_add`、`c.damage_critical_chance` 等：伤害与暴击。
- `c.extra_entities`、`c.game_effect_entities`：给后续投射物附加实体或状态。

投射修正器通常在 action 末尾调用 `draw_actions(1, true)`，表示“修改当前状态，然后把下一张法术纳入同一 shot”。如果没有后续投射物，很多修正器本身没有可见效果。

## 链锯为什么强

`CHAINSAW` 是典型底层特例：它生成一个寿命极短的切割投射物，同时把 `c.fire_rate_wait` 设为 0，并额外降低 `current_reload_time`。这不是普通“低延迟法术”，而是直接改写施法延迟和充能时间，所以在法杖包裹和多重释放中经常成为高速循环核心。

## 负法力法术

例如 `MANA_REDUCE`（Add Mana）声明 `mana=-30`。`draw_action()` 的法力检查会允许负法力通过，随后 `handle_mana_addition()` 这类逻辑会把负消耗转成法力增加。它仍然可以增加施放延迟或造成其他副作用，不能只看 mana 数值。

## 数值换算提醒

Noita 的 `c.fire_rate_wait`、`current_reload_time` 和许多 XML `lifetime` 都以帧为核心单位；游戏通常按 60 FPS 解释。MC-noita 运行在 Minecraft 20 TPS，需要统一换算，否则高速法杖和短寿命投射物会偏差很大。
