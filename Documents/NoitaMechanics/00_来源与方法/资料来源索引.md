## 自动提取方法

1. 读取 `gun_actions.lua` 中的 `actions = { ... }` 表。
2. 提取每个 action 的 `id`、`type`、`mana`、`max_uses`、`spawn_level`、`related_projectiles`、`custom_xml_file`。
3. 读取 `common.csv` 的英文和简体中文列，解析 `$action_*` 名称。
4. 从 `action = function()` 中提取 `add_projectile*`、`draw_actions()`、`c.*` 状态改写、`current_reload_time`、`mana` 和常见引擎调用，生成行为摘要。
5. 剔除 Lua 行注释，避免把注释掉的调试代码当成真实行为。

## 可信度边界

- 可以确认：卡牌 action 的公开 Lua 逻辑、action 类型、名称、法力、次数、生成池、常见投射物 XML。
- 基本可信但需谨慎：Wiki 对高级组合技的解释，如递归、包裹、手牌/弃牌细节。
- 无法完全确认：闭源 C++ 层 `BeginProjectile`、`BeginTrigger*`、`ConfigGunActionInfo_PassToGame` 的内部实现细节。本归档只根据 Lua 调用边界和可观察效果推断。
