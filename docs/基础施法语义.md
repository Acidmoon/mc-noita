# 基础施法语义（G02）

## 范围与证据

G02 固化纯法杖解释器的 Deck、Hand、Discard、法力、次数、Always Cast、Wrap、Reload 与时序语义。它不实现 Greek/Copy 的完整目标策略、递归规则、Trigger/Timer/Death 的实体生命周期或新的世界效果；这些保持 G03 及之后 Goal 的待完成项。

行为依据按项目优先级使用以下资料：

- Noita Wiki：<https://noita.wiki.gg/wiki/Expert_Guide:_Draw> 与 <https://noita.wiki.gg/wiki/Guide:_Wand_mechanics>，记录检索日期为 2026-07-14。
- 固定版本实现证据：`Documents/NoitaMechanics/_source_cache/noitadata/data/scripts/gun/gun.lua` 的 `draw_action`、`draw_actions`、`_handle_reload`、`_play_permanent_card` 与 `move_hand_to_discarded`。
- 固定动作证据：`Documents/NoitaMechanics/_source_cache/noitadata/data/scripts/gun/gun_actions.lua` 的 `MANA_REDUCE` 与 `CHAINSAW`。
- 本地 Wiki 摘要：`docs/noita法杖.txt`。其中记录预载时可临时超过 `manaMax`，预载完成后会削去溢出值。

纯解释器语义标记为 `faithful`。Minecraft 的帧到 tick 向上取整属于明确的 `equivalent` 边界，集中在 `MinecraftTimeAdapter`。

## Draw 状态机

`WandCastSession` 不再接收 `drawMany(amount, canWrap, permanent)`。每个请求由 `DrawRequest` 表示，并把来源写入不可变 `DrawOutcome`：

| `DrawOrigin` | 调用者 | 可发起 Wrap | 特殊规则 |
|---|---|---:|---|
| `INITIAL` | `wand.spellsPerCast()` | 否 | Deck 空时设置本次 `reloading`，之后不再抽牌 |
| `ACTION` | `DrawAction`、Multicast、Modifier | 是 | Deck 空且 Discard 非空时可 Wrap |
| `PAYLOAD` | Trigger 的预解析载荷 | 是 | 使用独立 Shot State，仍在本次施法支付法力 |
| `PERMANENT` | Always Cast 内 `DrawAction(n > 1)` | 是 | `n == 1` 被抑制，`n > 1` 是正常的额外抽牌 |

一次正常候选卡的顺序不可拆分：

1. 从 Deck 顶部移除 `CardRef`。
2. 检查剩余次数。
3. 查找定义并检查法力。
4. 仅在成功时扣除或增加一次法力。
5. 移入 Hand。
6. 执行该定义的 action。

次数耗尽、未知定义或法力不足的候选卡立即进入 Discard，不会进入 Hand 或执行 action。只要 Deck 中还有候选卡，同一普通 Draw 会继续尝试。最后一张候选卡失败时，当前请求直接结束，不能在同一请求里 Wrap 并再次抽取本轮的卡。

`DrawOutcome` 保存成功卡、失败卡及原因、Deck 是否耗尽、是否 Wrap、当前 `startReload` 和 `reloading`。它随 `ResolvedCast` 返回，供测试、诊断和之后的 UI 使用。

## Wrap、Reload 与刷新

会话内状态各自独立：

- `reloading`：一次 Draw 发现 Deck 空且该来源不能 Wrap。后续 Draw 停止。
- `startReload`：Action/Payload/Permanent Draw 已经 Wrap；本次施法结束仍须 Recharge。
- `refreshCount`：同一施法内已经生效的 Wand Refresh 次数。

| 转换 | 牌堆操作 | 是否 `startReload` | 是否开始 Recharge |
|---|---|---:|---:|
| Initial Draw 发现空 Deck | 不移动牌 | 否 | 施法结束时 Deck 为空则开始 |
| Action/Payload Draw Wrap | `Discard -> Deck -> order/shuffle`，继续当前请求 | 是 | 是 |
| 普通 cast 结束且 Deck 为空 | `Hand -> Discard -> Deck -> order/shuffle` | 无关 | 是 |
| `startReload=true` 的 cast 结束 | `Hand -> Discard`，合并剩余 Deck 与 Discard 后 `order/shuffle` | 已消费 | 是 |
| 第一次 Wand Refresh | 合并 Hand、Deck、Discard 后 `order/shuffle` | 否 | 否，仅刷新本身不强制 Reload |
| 第二次 Wand Refresh | 不再重组牌堆 | 是 | 是 |

非乱序法杖在 Wrap、Reload、Wand Refresh 都按原始槽位升序。乱序法杖只在初始装载、Wrap、Reload、Wand Refresh 等转换点使用 `CastRng` 的命名子流洗牌；每次普通 Draw 不洗牌。所有重排都只重排现有 `CardRef`，所以每张配置卡始终恰好位于 Deck、Hand 或 Discard 之一。

Reload 已在纯解释器内完成并写入 `WandState`。适配器在 Recharge 到期时只释放冷却，不再用下一次施法或 HUD 查询的 seed 重建 Deck。

## Mana 与 Uses

`SpellDefinition.manaCost` 可为负数。正常 Draw 对正法力只检查和扣除一次；失败抽牌不扣法力。`MANA_REDUCE` 的 `mana=-30` 通过相同的正常 Draw 路径先把法力加回，使同一 cast 后续卡可使用这部分法力。预载过程中允许临时超过 `manaMax`，但 `ResolvedCast` 在预载结束时将溢出夹回 `manaMax`。

自然恢复按 `elapsedFrames / 60 * manaChargePerSecond` 计算，并在恢复步骤夹到 `manaMax`。适配器用 `LastManaTick` 保存最后结算时间：法杖不在手上仍能在下一次读取/施法时恢复，而 HUD 只投影计算，不能重复写入恢复值。

有限次数在 Hand 移入 Discard 时结算，不在成功 Draw 时结算。`UseConsumptionPolicy` 明确每个定义的策略：

- `WHEN_PROJECTILE_SHOT`：本次正常 Hand 中存在 Projectile、Static Projectile 或 Material 卡时扣一次。
- `ALWAYS_ON_HAND_DISCARD`：Other/Utility 正常进 Hand 后扣一次。
- `NEVER`：Passive 或自定义次数逻辑不自动扣除。

因此失败 Draw、Always Cast、普通 Call 目标不会自动扣次数；同一张卡即使产生多个实体或多个 Trigger 也最多扣一次。

## Always Cast 与时序

Always Cast 先于 Initial Draw 执行，永不作为配置卡进入 Deck、Hand 或 Discard。正法力消耗视为零、次数视为无限；负法力仍按 `handle_mana_addition` 增加法力。永久附着卡内部的 `DrawAction(1)` 被抑制，`DrawAction(n > 1)` 仍创建 `PERMANENT` Draw 请求。Always Cast Trigger 的载荷在外层施法时完成预解析、支付其普通载荷法力并冻结到 `EffectPlan`；碰撞生命周期不属于 G02。

解释器保存 Noita 帧。Cast Delay 与 Recharge Time 从同一施法时刻并行倒计时，下一次施法须同时等待两者结束，即等待较大者。只有实际 Reload 才启动基础 Recharge；未 Reload 的 cast 只保存 Cast Delay。Trigger payload 的 Cast Delay 不影响外层，但其 Recharge 修改仍是全局值。`TimingAction` 通过 `ADD` 与 `SET` 表达累加与赋值，Chainsaw 的 `c.fire_rate_wait = 0` 由 `SET` 映射。所有负时间最终夹到零。

## 持久化与验证

Cast State schema 已升至 v2。v1 G01 状态会添加 `G02ReloadPrepared=false` 标记；遇到旧的“空 Deck、Discard 有卡、等待 Recharge”形态时，适配器使用 `SpellsHash` 与 revision 派生的固定迁移 seed 重建一次 Deck，而不是使用调用方随机数。新 G02 写入的 Deck 已经由接受本次 cast 的纯解释器决定，写入 `G02ReloadPrepared=true`。

主要验证入口：

```powershell
.\gradlew.bat test
.\gradlew.bat gameTest
.\gradlew.bat runGametest --no-daemon
.\gradlew.bat verifyG02
```

`G02WandCastSemanticsTest` 覆盖非乱序、固定 seed、Initial/Action Draw、末卡失败、Reload、Wand Refresh、正负法力、次数策略、Always Cast、并行冷却和属性循环。真实 Fabric GameTest 使用三枚 Spark Bolt 与 `Spells/Cast=2` 连续施放两轮，比较生产 `NoitaWandCaster` 的 Deck、法力、冷却和实际实体数与纯解释器输出。

G03 保持明确待办：Greek/Copy/Divide/Add Trigger 策略、递归标记与预算、Trigger/Timer/Piercing/Death 的实体持久化及 exactly-once 生命周期。
