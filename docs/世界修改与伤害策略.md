# 世界修改与伤害策略（G05-7）

## 责任边界

`com.mcnoita.world.mutation` 是服务端法术进行方块修改、危险实体生成、爆炸地形候选和大范围实体查询的唯一边界。它不解释卡牌、不扣除法力、不重算 Trigger 载荷，也不定义 Noita 的伤害通道；这些仍分别属于 evaluator、事务协调器和后续 `DamageProfile` 工作。

当前入口是：

```text
SparkBoltProjectileEntity / BombEntity
  -> WorldMutationContext
  -> WorldQueryService 或 WorldMutationService
  -> WorldMutationPolicy
  -> Minecraft ServerWorld
```

`WorldMutationContext` 绑定 source、owner UUID、预期维度、冻结 execution identity 和预算钩子。执行时重新从服务器 PlayerManager 解析 owner，并要求 owner 在线、未移除、存活且仍在同一维度；因此 owner 登出、死亡、跨维度或缺失 owner 都不能继续进行破坏性操作。

## 默认保守策略

破坏性方块操作、爆炸和危险实体生成必须同时满足：

- 逻辑服务端、预期维度、build limit、world border 和目标 chunk 已加载。
- owner 是当前在线且仍在同一 `ServerWorld` 的玩家。
- `mobGriefing` 为 true。G05 明确把它应用于玩家施法、召唤、反射投射物和离线 owner 的遗留投射物。
- spawn protection 与 `ServerWorld.canPlayerModifyAt` 均允许。
- 所有已注册 `ProtectionAdapter` 都允许。
- 方块不在 `mc-noita:spell_unbreakable`，硬度有限、非负且不高于 20。
- 预算钩子接受本次检查、修改、生成或查询。

没有 claims adapter 时不会声称通用领地兼容；默认仍由原版 spawn protection 和上述保守拒绝路径保护。

`data/mc-noita/tags/blocks/spell_unbreakable.json` 保护基岩、屏障、命令/结构方块、末地传送方块和强化深板岩。服务器数据包可扩展该 tag。

## 有界操作

`WorldQueryService` 要求整个 query Box 的 chunk 已加载，最大直径为 64、最大体积为 262,144、最多跨 25 个 chunk，结果最多 128 个实体；几何上限会在任何 chunk 枚举之前检查。`WorldMutationService.explode` 限制半径为 16，先使用无火的 `ExplosionSourceType.NONE` 保留原版实体击退/伤害表现而禁止原版地形扫描，再最多枚举 512 个球形方块候选，并逐个经过策略和预算后破坏。原版 explosion fire 被强制关闭，法术放火必须逐块走 `replaceBlock`。

这属于 Minecraft 等效实现，不承诺逐像素 Noita 爆炸地形。后续中心 `SpellBudgetManager` 应通过 `WorldMutationBudget` 替换当前默认未跟踪钩子，使 per-cast、owner、chunk、dimension 和 global reservation 共享同一额度。

## 现有保护与限制

Trigger v3 的 frozen payload、runtime budget、execution identity 和正常 save/reload at-most-once 语义保持不变。该策略不把普通副作用升级为 crash-safe exactly-once，也不创建跨 tick job；这些属于 G05-8。

伤害仍由旧 `NoitaSpellDamage` 和 `indirectMagic` 适配，不在本工作包变更。后续正式伤害模型必须把直接伤害、治疗和状态效果移出实体分支。

## 测试与扩展

- `WorldMutationBudgetCounterTest` 验证纯预算钩子不会在失败时部分扣除。
- `WorldMutationArchitectureTest` 与 `checkMainServerArchitecture` 阻止 Spark/Bomb 直接调用危险世界 API。
- 未来 adapter 应只实现 `ProtectionAdapter.allows`，不得从 adapter 强加载 chunk 或修改世界。
- 新增世界副作用必须先选择 `WorldQueryService` 的有限查询，再调用 `WorldMutationService`，不能回到实体内的 `World` 直接调用。
